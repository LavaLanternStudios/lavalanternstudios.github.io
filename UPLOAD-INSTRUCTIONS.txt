FRONTIER-STYLE HERO VIDEO FIX

WHY THIS CHANGE IS NEEDED

The visible “This video is unavailable” message is generated by YouTube.
It cannot be corrected with autoplay parameters or CSS.

This package replaces YouTube hero iframes with ordinary HTML5 background
videos. Normal YouTube walkthrough videos inside portfolio-page content are
not changed.

STEP 1 — REPLACE THE CODE FILES

Copy the contents of this package into the root of the cloned repository.
Allow all matching files to replace the existing versions.

The package updates:

style.css

assets/content/home-page.js
assets/content/games-page.js
assets/content/fab-page.js
assets/content/portfolio-page.js

assets/js/home-content.js
assets/js/games-content.js
assets/js/fab-content.js
assets/js/portfolio-content.js

portfoliopages/portfoliostyle.css
portfoliopages/assets/data/portfoliopage-loader.js
portfoliopages/assets/data/portfoliopages.js


STEP 2 — ADD THE FOUR MAIN-PAGE MP4 FILES

Create this folder in the repository:

assets/videos/

Add these files with these exact names:

assets/videos/home-hero.mp4
assets/videos/games-hero.mp4
assets/videos/fab-hero.mp4
assets/videos/portfolio-hero.mp4


STEP 3 — ADD THE SIX PORTFOLIO HERO MP4 FILES

Add each file inside the existing matching project folder:

portfoliopages/assets/projects/lifeguard-sim/hero.mp4
portfoliopages/assets/projects/prison-level/hero.mp4

STEP 4 — PREPARE THE VIDEOS

Recommended hero-video preparation:

- MP4 container
- H.264 video
- 1920 × 1080 or 1280 × 720
- 10–30 seconds where practical
- Remove the audio track
- Use a clean visual loop
- Keep every file comfortably below 100 MiB
- Around 5–20 MiB per hero is substantially better for loading speed


STEP 5 — COMMIT AND PUSH

Open GitHub Desktop.

Commit summary:
Replace hero embeds with background videos

Click:
Commit to main
Push origin

Wait for GitHub Pages to deploy, then refresh with Ctrl + F5.


RESULT

All ten configured hero videos will:

- autoplay
- remain muted
- loop continuously
- play inline on mobile
- fill the hero using object-fit: cover
- show no YouTube controls, branding or unavailable-message overlay
- retry playback after returning to the tab or restoring the page


IMPORTANT

The MP4 filenames and locations must match this checklist exactly.
The code cannot display a video until its corresponding MP4 file has been
placed in the repository.
